# このファイル一式について

レスポンスの記事から正確にショート動画用のストーリースクリプトを生成するプロンプトと、さらにそこから生成されたスクリプトをファクトチェックさせるプロンプトの2本です。

## Markdownについて

今回のプロンプトファイルはMarkdownという形式で書かれています。Markdownファイル（.md）はシンプルなテキストファイルなので基本的には.txtファイルと一緒です。見出しや強調、リストなどの書き方が決まっているだけです。AIにプロンプトで複雑な指示をするときはMarkdown方式で書くと正確に意図を伝えることができます。
AI関連の業界では標準的なフォーマットとなっています。

AI向けのプロンプトとしてよく使うのは以下です。

- 見出し （#、##、###、……）
- 強調（*〜*）
- リスト（行頭の- 、*、など）
- コードブロック（データを渡すときに使います。```〜```）

Markdownについてより詳しくは以下のサイトを参照してください。
https://www.tohoho-web.com/ex/markdown.html

## テキストエディタについて

Markdownファイルを複数扱うときはテキストエディタがあると便利です。
Windowsの「メモ帳」がもっともシンプルなテキストエディタですが、以下が一般的です。
- VS Code （Visual Studio Code）：マイクロソフト製高性能エディタ。無料で業界標準ですが、高機能すぎ重い。
  - https://code.visualstudio.com/download
- Zed：VS Codeライクな軽量高速エディタ。無料ですが日本語化はできない。
  - https://zed.dev/
- CotEditor：長い歴史を持つ国産テキストエディタ
  - https://coteditor.com/

## AI向けのプロンプトの書き方

AIのプロンプトの書き方は「プロンプトエンジニアリング」という、専門的な技術になっています。
ある意味で「自然言語（日本語）によるプログラミング」であるとも言えます。

プロンプトエンジニアリングの基本は以下です。すでに知っていることと思いますが改めて基本に忠実にやってみてください。

1. 具体的に頼む
「いい感じの文章を書いて」ではなく、「社内向けのお知らせメールを、丁寧だけど堅すぎないトーンで、200字程度で書いて」のように伝えましょう。誰に向けたものか、どんな目的か、どのくらいの長さか。人に仕事を頼むときと同じで、情報が多いほど結果が良くなります。
2. 背景情報を渡す
AIはあなたの状況を知りません。「この企画書を直して」ではなく、「これは中小企業の経営者向けのIT導入提案書です。専門用語を減らして読みやすくしてください」と背景を添えると、的確な回答が返ってきます。
3. お手本を見せる
「こんな感じで書いて」と例を1つ見せるだけで、出力の質が大きく変わります。過去に自分が書いた文章や、理想に近いサンプルを貼り付けて「このトーンに合わせて」と伝えるのが効果的です。
4. 一度で完璧を求めない
最初の回答が70点でも、「もう少しカジュアルに」「3番目の段落をもっと短く」と追加で指示すれば精度が上がります。AIとの対話は一発勝負ではなく、キャッチボールです。
5. 最新モデルなら「シンプルに頼む」が正解
以前は「ステップバイステップで考えて」のような特殊な言い回しが効果的でしたが、2025年現在の最新モデルでは、普通の日本語で明確に頼めば十分です。小手先のテクニックより、何をしてほしいかを過不足なく伝えることのほうがずっと大事です。

## プロンプトはAIに書いてもらう

今回のスクリプト作成プロンプトとファクトチェックプロンプトは両方とも生成AIのClaude Opus 4.6 に書いてもらったものです。AIに複雑な作業を頼むときは、的確なプロンプトをAIに書いてもらうのが効率的です。

### スクリプト作成プロンプトの作成依頼

```
自動車メディアレスポンスのニュース記事を30秒のTikTokのショート動画にするプロジェクトを行っています。生成AIで記事の内容を要約して、動画用のスクリプトを生成するためのプロンプトを考えてください。ポイントは名称や数値データなどを正確に動画化することです。

動画化する記事のサンプルは以下です。
https://response.jp/article/2026/02/23/407755.html
https://response.jp/article/2026/02/25/407881.html
```
