# OpenClawのCPU・メモリ要件

**OpenClawのCPU・メモリ要件について、2026年2月現在の最新実測情報でしっかりまとめました！**

公式GitHub/READMEおよび公式サイト（openclaw.ai）では**明示的な最小スペックは記載されていません**（「Node.js 22以上でmacOS/Linux/Windows(WSL2)/Raspberry Piで動く」とだけ）。  
しかし、**コミュニティの実測・VPSガイド・DigitalOcean公式テンプレート**など数百の報告から、はっきりした目安が出ています。

### 公式・コミュニティ共通の結論
- **「最低限2GB」と言われる理由**：インストール自体は1〜2GBで可能（npmインストール時にswapを使えば動く）。  
  → でも**すぐに不安定**になり、ブラウザ自動化や複数スキル起動でクラッシュしやすい。
- **アシスタントやAI社員として「実際に使う」なら**：**最低4GB、快適に8GB以上**を強くおすすめします。

### 用途別おすすめスペック表（クラウドLLM使用前提）
| 用途レベル                  | RAM     | CPU / vCPU       | ストレージ | 実用感・おすすめ環境                          | 月額目安（VPS） |
|-----------------------------|---------|------------------|------------|-----------------------------------------------|-----------------|
| **絶対最低（テストのみ）**  | 2GB    | 1〜2コア        | 10GB+     | インストールOKだが遅い・不安定               | $3〜5          |

| **本格AI社員**<br>（マルチエージェント・ブラウザ多用・Zenn自動投稿・24/7 cron） | **8GB** | **4コア以上**   | 50GB+ SSD | **これが現実的なおすすめライン**<br>ほぼ止まらない | $8〜15         |
| **ヘビーユース**<br>（10エージェント以上・大規模自動化） | 16GB+  | 8コア以上       | 100GB+    | Mac mini M3/M4 16〜32GBが最強                | 専用機推奨     |

### 実際のメモリ・CPU消費イメージ（クラウドLLM使用時）
- **アイドル時**：100〜500MB RAM、ほぼ0% CPU（Gateway本体は超軽量）
- **通常タスク時**（メール要約・ファイル整理）：1〜2GB程度
- **重いタスク時**（ブラウザ自動操作・複数スキル同時）：一時的に**3〜6GB**まで跳ね上がる
- 主な消費要因：Headless Chrome、MEMORY.md読み込み、Dockerサンドボックス（オプション）、cron同時実行

**※ローカルLLM（Ollamaなど）を使う場合**  
→ 上記に**+8〜24GB以上**必要（例：Qwen3-30Bなら最低16GB推奨）。  
クラウドLLM（Claude/Grok/GPT）だけなら表の通りでOKです。

### みんなが実際に使っている人気構成（2026年現在）
1. **一番おすすめ**：**Mac mini M3/M4 16GB**  
   → unified memoryのおかげで効率抜群、低電力（15W前後）、24/7で熱くならない。コミュニティで「人生変わった」報告多数。
2. **コスパ最強**：**Raspberry Pi 5 8GB**（約1万円）  
   → 軽いアシスタントなら余裕。Pi 4 4GBでも動くが8GB推奨。
3. **VPSで手軽**：DigitalOcean / Hetzner / Oracle Cloudの**4〜8GBプラン**（$5〜10/月）  
   → Ubuntu 24.04で1コマンドインストール。Oracleの無料枠（24GB）でも余裕。

### 私のアドバイス（あなたの使い方向け）
- **まずは今お持ちのMac/PCで試す**：4GB以上あれば十分動きます。  
  動かしてみて「遅いな」と感じたら専用機に移行すればOK。
- **本気でAI社員として24/7運用したい**なら → **8GB以上**を最初から用意してください。  
  後からメモリ増設できない環境（RPiや安いVPS）だと後悔します。
- ストレージはSSD推奨（MicroSDでも動くが遅い）。

実際に多くの人が「4GBで十分だけど、8GBにしてから本領発揮した」と報告しています。
