# OpenClaw専用機器：4つの選択肢（修正版）

本稿は、公式ドキュメントで確認できる要件・注意点に基づき、**価格やベンチマークなど変動の大きい数値は固定せず**に比較する。価格や消費電力、ベンチマークは時期と構成で大きく変わるため、最新情報は各製品・サービスの公式ページで確認すること。

---

## OpenClawの公式要件（確認済み）

- **Node.js 22以上が必須**（インストーラが未導入なら自動で導入する）。
- **OSは macOS / Linux / Windows（WSL2推奨）**に対応。
- **Dockerは必須ではない**（コンテナ運用などの場合に使用）。
- ソースからビルドする場合は `pnpm` が必要。

---

## リソースの目安（FAQベース）

公式FAQの記載を基に、最小・推奨の目安をまとめる。

- **最小構成（Gateway + 1チャネル想定）**: 1 vCPU / 1GB RAM / 約500MBのディスク
- **推奨**: 1〜2 vCPU / 2GB以上のRAM（ログ・メディア・複数チャネルの余裕）
- **Raspberry Pi**: 512MB〜1GB RAM、1コア、約500MBのディスクでも個人用途なら動作可能。余裕を持たせるなら **2GB** 推奨。
- **BunはGateway用途では推奨されない**。

## 4つの選択肢（定性比較）

### 1. Mac mini（M1/M2 など）

**向いている人**
- macOS専用ツール（例: iMessage連携など）を使いたい
- 静音・省電力の常時稼働機が欲しい

**注意点**
- メモリ・ストレージは購入時の構成固定になりやすい
- Mac miniは必須ではない（Mac以外でも動作可能）

### 2. Windows ミニPC（x86）

**向いている人**
- 価格を抑えつつメモリやストレージを後から拡張したい
- Linuxをネイティブ、またはWSL2で運用したい

**注意点**
- Windows上で動かす場合は **WSL2推奨**

### 3. Raspberry Pi 5（またはPi 4）

**向いている人**
- 省電力で小型の常時稼働機が欲しい
- Gateway用途を最小構成で試したい

**注意点**
- 性能・I/Oは一般的なPCより低くなりがち
- 複数チャネルや重い自動化を想定する場合は余裕のある構成が望ましい

### 4. クラウドVPS

**向いている人**
- 自宅に常時稼働機を置きたくない
- 可用性やリモート運用を重視したい

**注意点**
- 有料VPSは月額コストが継続的に発生
- 無料枠は条件が変わる可能性があるため、公式の利用条件を必ず確認する
- Oracle Cloud Always Free などでは、**アイドル状態が続くとインスタンスが回収される**条件がある

## チャネル対応（参考）

OpenClawはGateway経由で複数のチャネルに接続できる。代表的な対応チャネルは以下。

- WhatsApp
- Telegram
- Discord
- Slack
- LINE（プラグイン）
- Google Chat
- Signal
- iMessage（macOS側が必要）

**補足**: 一部チャネルはプラグインを別途インストールする必要がある。チャネルは複数同時に動作可能。

---

## まとめ：選択の目安

- **最小コストで試す**: Raspberry Pi or 小型VPS
- **安定稼働・拡張性重視**: x86ミニPC or 有料VPS
- **macOS専用機能を使う**: Mac mini

要件は比較的軽量だが、**複数チャネル運用やブラウザ自動化を想定する場合は余裕のある構成が安全**。用途に合わせて「常時稼働できるか」「拡張できるか」「運用コストは許容できるか」を基準に選定するのが現実的。
