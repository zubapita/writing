# 日本語文字化け対策

日本語が化けない理由：4層構造

  Layer 1: プロンプト冒頭の強制指示

  lib/services/image-generation-service.ts で、以下の指示がプロンプト冒頭に必ず挿入されます：

  ENSURE ALL TEXT RENDERED IN THE IMAGE IS JAPANESE ONLY. NO ENGLISH.

  これが2段階で保証されています：
  - suggestPrompt() でAIにプロンプト生成させる際、この制約を含めるよう指示
  - generateImage() で、万が一含まれていなければ自動で先頭に追加

  Layer 2: 具体的な日本語文字列の直接指定

  プロンプト内でラベルやタイトルを英語で記述せず、具体的な日本語文字列をexactly:形式で指定しています：

  The main title must be exactly: "生成AIのビジネス活用"
  Labels must be: "データ収集", "分析", "実装"

  これにより、AIが「翻訳」する余地がなく、そのままの日本語が画像に描画されます。

  Layer 3: CSSでの日本語フォント優先指定

  app/globals.css で日本語フォントを最優先に指定：

  --font-sans: "Noto Sans JP", "Hiragino Sans", "Yu Gothic", system-ui, sans-serif;
  font-feature-settings: "palt" 1;  /* 日本語用プロポーショナル調整 */

  Layer 4: 検証→再生成ループ

  Gemini VisionとGPTの2系統で生成画像を検証し、不正確な日本語があれば日本語フィードバック付きで再生成します（最大2回リト
  ライ）。

  ---
  他システムで再利用するための核心ポイント

  他のプロジェクトで同じ品質を実現するには、以下の3つが最重要です：

  1. プロンプト冒頭に ENSURE ALL TEXT RENDERED IN THE IMAGE IS JAPANESE ONLY. NO ENGLISH. を必ず入れる
  2. 画像内テキストは英語で説明せず、具体的な日本語文字列をexactly:形式で直接指定する
  3. Gemini gemini-3-pro-image-preview モデルを使う（このモデルはプロンプト内の言語制約を比較的忠実に守る）

  プランファイルに書き出します。

  まとめ：日本語が化けない秘訣

  最も効果的な要素は以下の3つです：

  1. プロンプト冒頭の英語禁止宣言

  ENSURE ALL TEXT RENDERED IN THE IMAGE IS JAPANESE ONLY. NO ENGLISH.
  image-generation-service.tsのgenerateImage()で、インフォグラフィクスの場合にこの文言が含まれていなければ自動で先頭に追
  加するフェイルセーフが入っています。

  2. exactly: 形式での具体的な日本語文字列指定

  AIに「翻訳」や「判断」の余地を与えず、画像に描画すべき日本語を直接指定しています。例：
  The main title must be exactly: "生成AIのビジネス活用"
  Labels must be: "データ収集", "分析", "実装"

  3. 検証→再生成ループ

  Gemini VisionとGPTの2系統で画像内テキストを検証し、問題があれば日本語フィードバック付きで最大2回再生成します。

  ---
  他のシステムに最低限移植するなら、プロンプト冒頭のNO ENGLISH宣言 と exactly:形式での日本語直接指定
  の2つが最も即効性があります。
