# OpenClawのマルチチャネル対応について

---

## アーキテクチャの基本

OpenClawの中核にあるのは**Gateway**という単一プロセスです。すべてのメッセージングプラットフォームからの入出力はこのGatewayに集約され、エージェントのロジック自体はチャネルに依存しません。各プラットフォームは**チャネルアダプター**としてプラグイン化されており、メッセージは共通フォーマットに正規化されてからエージェントに渡されます。

つまり「WhatsAppで聞いたことの続きをTelegramで尋ねる」といったクロスチャネル利用でも、エージェント側のコンテキストは保持されます。

---

## 対応チャネル一覧

公式ドキュメントとGitHubで確認できる対応チャネルは以下の通りです。

**メインチャネル（公式サポート）**

- WhatsApp — QRコードでWhatsApp Web経由で接続。Baileys（非公式ライブラリ）を使用
- Telegram — BotFather経由でボットトークンを取得。最もセットアップが簡単で公式推奨
- Discord — Bot API経由。サーバー/チャンネル/DM対応。コミュニティ用途に強い
- Slack — Boltフレームワーク＋OAuth。企業・チーム向け
- Signal — signal-cli経由。エンドツーエンド暗号化で最もプライバシーが高いが設定は複雑
- iMessage — macOS上のMessages.appオートメーション経由（レガシー方式）
- BlueBubbles — iMessageの拡張チャネル（macOS不要の方式も）
- Microsoft Teams — Bot Framework＋Azure Portal。エンタープライズ向け
- Google Chat — Workspace統合
- WebChat — ブラウザ上の組み込みUI

**拡張チャネル**

- Matrix（オープンプロトコル）
- Zalo / Zalo Personal（ベトナムで主流のメッセンジャー）
- IRC
- Feishu（Lark/飛書 — 中国・アジア圏のビジネスチャット）
- Mattermost（セルフホスト型チームチャット）
- Synology Chat
- LINE
- Nextcloud Talk
- Nostr（分散型SNSプロトコル）
- Tlon（Urbit系メッセンジャー）
- Twitch

**コンパニオンアプリ・音声**

- macOSメニューバーアプリ（Gateway制御＋Voice Wake）
- iOS / Androidノード（Bridge経由でペアリング、Canvas・カメラ・画面キャプチャ対応）
- Voice Wake + Talk Mode — 「Hey OpenClaw」で音声起動、ElevenLabsによるSTT/TTS

## 各チャネルの特徴と選び方

**Telegram**が最も手軽です。BotFatherでトークンを発行するだけで、QRスキャンもOAuthも不要。ロングポーリングがデフォルトで動作し、ボイスノート対応、リッチフォーマット、インラインキーボード、リアクションなど機能が最も充実しています。ただしメッセージはデフォルトではE2E暗号化されません。

**WhatsApp**は世界的に利用者が多く、スマホからの操作が最も自然ですが、非公式ライブラリ（Baileys）経由のためプロトコル更新時に不安定になるリスクがあります。

**Discord**はコミュニティ運用やチーム開発に向いています。スレッド機能でサブエージェントの長時間タスクを追跡するような使い方が便利です。個人用途としてはやや大げさです。

**Slack / Microsoft Teams**は企業環境でのデプロイに適しています。スレッド、リアクション、管理者制御が充実しています。

**Signal**はプライバシー最優先のユーザー向けですが、signal-cliのセットアップと暗号鍵管理が必要で、導入ハードルは高めです。

## セキュリティとアクセス制御

チャネルごとに細かいアクセス制御が用意されています。

**DMポリシー** — デフォルトは「pairing」で、初回接続時にペアリングコード（1時間有効）を要求します。これにより、未承認ユーザーがボットに話しかけても応答しません。

**allowFrom / groupAllowFrom** — ユーザーIDベースのホワイトリスト。Telegramなら数値ID、WhatsAppなら電話番号で指定します。

**グループポリシー** — `groupPolicy: "allowlist"`で許可グループのみ応答。`requireMention: true`にすればグループ内でボットをメンションしたときだけ反応する設定も可能です。

`openclaw doctor`コマンドで、危険なDMポリシー設定がないか自動診断もできます。

## 実際の使い方のイメージ

OpenClawをVPSやMac Miniで常時稼働させておくと、以下のような使い方が可能になります。

外出先でスマホのTelegramからエージェントに「今日のメール要約して」と送る → エージェントがメールを確認して要約を返信 → 帰宅後にPCのDiscordで「さっきのメールの件、返信案を作って」と続きを依頼 → エージェントはTelegramでの文脈を覚えているのでシームレスに対応

このように、OpenClawは「AIアシスタント専用の新しいアプリを入れる」のではなく、**普段使っているメッセージングアプリがそのままAIへのインターフェースになる**という設計思想です。これがChatGPTやClaudeのWebUIとの根本的な違いであり、「いつでもどこからでも、使い慣れたアプリで」AIに指示を出せる利便性につながっています。
